Les types de données int, char, float et double en C++ (et en C) sont encodés de manière différente en raison de leur nature et des besoins spécifiques en matière de représentation et de précision. Voici les principales différences d'encodage entre ces types :
1. int (Entier)

    Représentation : Les entiers sont généralement stockés en utilisant une représentation en complément à 2, ce qui est une méthode courante pour représenter les nombres entiers signés en binaire.
    Taille : La taille d'un int peut varier en fonction de l'architecture de la machine et du compilateur, mais elle est généralement de 4 octets (32 bits) sur la plupart des systèmes modernes. Certains systèmes peuvent utiliser 2 octets (16 bits) ou 8 octets (64 bits).
    Plage de Valeurs :
        Pour un int de 4 octets, la plage typique est de -2,147,483,648 à 2,147,483,647 (pour les entiers signés).
        La plage pour les entiers non signés (comme unsigned int) est de 0 à 4,294,967,295.

2. char (Caractère)

    Représentation : Les caractères sont stockés en utilisant un encodage binaire qui représente chaque caractère dans un ensemble de caractères (comme ASCII ou UTF-8).
    Taille : Un char est typiquement de 1 octet (8 bits) sur la plupart des systèmes.
    Plage de Valeurs :
        Pour un char signé, la plage est généralement de -128 à 127.
        Pour un char non signé (unsigned char), la plage est de 0 à 255.

3. float (Flottant Simple Précision)

    Représentation : Les float sont représentés en utilisant le format IEEE 754 pour les nombres flottants en simple précision. Ce format utilise 32 bits pour stocker les valeurs :
        1 bit pour le signe
        8 bits pour l'exposant
        23 bits pour la mantisse (ou significand)
    Précision : Les float ont une précision d'environ 6 à 7 chiffres significatifs.
    Plage de Valeurs : La plage typique est d'environ ±3.4×1038±3.4×1038.

4. double (Flottant Double Précision)

    Représentation : Les double utilisent également le format IEEE 754, mais en double précision avec 64 bits :
        1 bit pour le signe
        11 bits pour l'exposant
        52 bits pour la mantisse
    Précision : Les double ont une précision d'environ 15 à 16 chiffres significatifs.
    Plage de Valeurs : La plage typique est d'environ ±1.7×10308±1.7×10308.

Résumé des Différences

    Type int : Stocke des nombres entiers, généralement 32 bits, utilisant la représentation en complément à 2.
    Type char : Stocke des caractères, généralement 8 bits, utilisant des encodages de caractères comme ASCII.
    Type float : Stocke des nombres à virgule flottante en simple précision (32 bits) avec une précision de 6 à 7 chiffres significatifs.
    Type double : Stocke des nombres à virgule flottante en double précision (64 bits) avec une précision de 15 à 16 chiffres significatifs.
